## 1. 개요

### 1.1 목적과 범위, 목표

#### 1.1.1 목적

본 문서는 연구실 안전관리 시스템 내 **안전교육 관리(SS-EDU)** 서브시스템에 대한 시험 계획과 시험 케이스를 정의한다. 구현된 안전교육일지 등록 및 조회 기능이 설계서(SDD)의 요구사항을 충족하는지 검증하는 것을 목적으로 한다.

#### 1.1.2 범위

본 문서의 시험 대상은 다음 두 유스케이스로 한정한다.

- UC-EDU-004: 안전교육일지 등록
- UC-EDU-007: 안전교육일지 조회

#### 1.1.3 목표

| 시험 계획 | 완료 기준 |
|---|---|
| 단위 시험 | SafetyJournalService의 createJournal, readJournalList 메서드가 정의된 모든 케이스에서 기대 결과를 반환한다 |
| 통합 시험 | Servlet → Service → Repository → JSON 파일 전 계층이 연결되어 정상 데이터 흐름을 보인다 |
| 시스템 시험 | 브라우저 기반 E2E 흐름(로그인 → 등록 → 목록 확인)이 오류 없이 완료된다 |
| 인수 시험 | 권한별 접근 제어 및 중복 차단이 실사용 시나리오에서 정상 동작한다 |

### 1.2 핵심 용어 및 약어

#### 1.2.1 용어

| 용어 | 정의 |
|---|---|
| 안전교육일지 | 연구실 안전교육 실시 내역을 기록한 문서. 교육 일자, 연구실, 교육 내용, 참석자 수, 작성자 정보를 포함한다 |
| 중복 등록 | 동일 연구실 ID와 동일 교육 일자의 안전교육일지가 이미 존재하는 상태에서 재등록을 시도하는 행위 |
| 형식 검사(FormalInspection) | 필수 항목 누락 여부 및 날짜 형식(yyyy-MM-dd) 유효성을 검사하는 절차 |
| 중복 검사(DuplicateInspection) | 동일 날짜·연구실 조합의 기존 데이터 존재 여부를 검사하는 절차 |

#### 1.2.2 약어

| 약어 | 정의 |
|---|---|
| SS-EDU | Safety Education Subsystem. 안전교육 관리 서브시스템 식별자 |
| STP | Software Test Plan. 소프트웨어 시험 계획서 |
| STD | Software Test Description. 소프트웨어 시험 기술서 |
| UT | Unit Test. 단위 시험 |
| IT | Integration Test. 통합 시험 |
| ST | System Test. 시스템 시험 |
| AT | Acceptance Test. 인수 시험 |
| UC | Use Case. 유스케이스 |

## 4. 시험 계획

### 4.1 단위 시험 계획

#### 4.1.1 목적 및 정의

단위 시험은 SafetyJournalService의 각 메서드가 독립적으로 올바르게 동작하는지 검증한다. 입력 데이터에 따른 정상 처리, 예외 발생, 경계 조건을 확인한다.

#### 4.1.2 단위 시험 대상

| 유스케이스 식별자 | 업무 구분 | 유스케이스 | 컴포넌트 ID | 단위 시험 ID |
|---|---|---|---|---|
| UC-EDU-004 | 안전교육 관리 | 안전교육일지 등록 | SS-EDU | UT-EDU-001 |
| UC-EDU-007 | (병합) | 안전교육일지 조회 | SS-EDU | UT-EDU-002 |

#### 4.1.3 세부 접근 방법

| 단위 시험 ID | 시험 절차 | 검증 기준 | 합격 기준 |
|---|---|---|---|
| UT-EDU-001 | SafetyJournalService.createJournal() 직접 호출 | formalInspection → duplicateInspection → insert 순서로 실행됨 | 정상 입력 시 true 반환, 오류 입력 시 적절한 예외 발생 |
| UT-EDU-002 | SafetyJournalService.readJournalList() 직접 호출 | 조건 파라미터(startDate, endDate, labId)에 따라 필터링된 목록 반환 | 조건에 맞는 데이터만 반환, 조건 없으면 전체 반환 |

#### 4.2.3 세부 접근 방법

| 단위 시험 ID | UT-EDU-001 | 시험 케이스명 | 안전교육일지 등록 |
|---|---|---|---|
| **시험 절차** | **입력 데이터** | **검증 기준** | **합격 기준** | **시험 결과** |

| TC-04 중복 등록: TC-01 완료 후 동일 labId·educationDate로 재등록 시도 | labId=LAB-001, educationDate=2026-06-19, content=다른 내용, attendeeCount=3, authorId=safetyteam | DuplicateDataException 발생, 메시지 "동일 교육일자·연구실의 안전교육일지가 이미 존재합니다." | 지정 예외 및 메시지 일치 | |
